> Команды <Win> CMD:
"DISK_NAME:PATH>" — путь к директории текущей области выполнения.
Область выполнения — это ограничение области видимости для исполняемых команд, которые от неё зависят. Создание файлов и работа с ними будет работать относительно этой точки, если в исполняемом коде или команде указаны относительные пути(относительно области выполнения).

:: directory
	cd <PATH> — переместить область выполнения в заданную директорию.
	dir       — показать содержимое директории области выполнения.
				(содержимое каталога)


> Команды Git:
:: initialization
	git init — инициализация репозитория в текущей области выполнения. 
				(подготовка к работе)
	
:: work
	git add <FILE_NAME / .(ALL)>   — добавить изменения в индекс. 
										(временное изменение)
	git status — проверить текущее состояние индекса.
					(проверка работы)
	git commit — сохранить(создать чекпоинт) изменений. Флаг -m "MESSAGE" — добавление сообщения(названия) коммиту.

:: sending
	git push                                     — отправка изменений на удаленный репозиторий.
	git push --set-upstream origin <BRANCH_NAME> — приатачить и отправить изменения ветки к удаленному репозиторию.
	git pull                                     — взять актуальные изменения.
												(скачать последние изменения из репозитория)
	git remote add origin <HTTP>                 — синхронизировать удаленный репозиторий к локальному по ссылке.

:: branches
	git branch                        — показать список веток.
	git branch <new.BRANCH_NAME>      — создание новой ветки изменений.
	git switch/checkout <BRANCH_NAME> — переключение на выбранную ветку.


> Команды Python:
:: venv
	python -m venv <new.VENV_DIR_NAME, default=venv>         — инициализация виртуального окружения пайтона внутри текущей области выполнения.
		(Она нужна для локальной установки зависимостей конкретного проекта. Иногда это важно, если они отличаются от актуальных версий. Или, например, разные проекты могут использовать разные версии зависимостей, что может привести к конфликтам, устанавливая зависимости глобально)
	<Win>           <VENV_DIR_NAME>\Scripts\activate         — активация виртуального окружения.
	<Linux / MacOs> source <VENV_DIR_NAME>/bin/activate      — активация виртуального окружения.
	pip install -r <TXT_FILE_NAME, default=requirements.txt> — установка всех нужных зависимостей из файла с их перечнем.
